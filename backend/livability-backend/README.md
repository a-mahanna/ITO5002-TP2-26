# Livability.AI Backend

FastAPI backend serving Melbourne suburb livability data.

## Setup

```bash
cd livability-backend
pip install -r requirements.txt
python -m uvicorn main:app --reload --port 8000
```

API docs at: http://localhost:8000/docs

## API Endpoints

### V1 — Suburb Lookup

| Endpoint | Description |
|----------|-------------|
| `GET /api/v1/suburbs` | List all suburb names |
| `GET /api/v1/suburbs/all` | Full data for all suburbs (map) |
| `GET /api/v1/suburbs/{name}` | Lookup a specific suburb |
| `GET /api/v1/search?q=foot` | Search suburbs by name |
| `GET /api/v1/averages` | Melbourne-wide averages |
| `GET /api/v1/compare?suburbs=Footscray,Brunswick,Richmond` | Compare up to 3 suburbs |

### V2 — AI Features

| Endpoint | Description |
|----------|-------------|
| `GET /api/v2/explain/{name}?budget=500` | Explainability engine |
| `GET /api/v2/similar/{name}?n=5` | Find similar suburbs |
| `GET /api/v2/recommend?budget=500&safety_weight=0.7&transport_weight=0.3` | Recommend suburbs |
| `GET /api/v2/clusters` | K-means suburb clusters |

## For Ali's Frontend

The Vue.js frontend should call these endpoints. Example:

```javascript
// Search suburbs
const response = await fetch('http://localhost:8000/api/v1/search?q=foot');
const data = await response.json();

// Get suburb details
const suburb = await fetch('http://localhost:8000/api/v1/suburbs/Footscray');

// Get all data for map
const all = await fetch('http://localhost:8000/api/v1/suburbs/all');
```

## Data Files

- `data/master_suburb_data_fixed.csv` — Raw merged CSV
